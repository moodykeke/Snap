# 多语言风格指南（精简版）

## 翻译原则
- 准确性优先于直译；保持术语一致（如“积木”“舞台”“调色板”）。
- 避免改变交互语义：保持占位符 `_`、参数引用 `<#n>` 数量与顺序。
- 保留大小写与标点风格：键名中 `...` 作为菜单省略，中文化后保留 `...`（不使用中文省略号）。

## 文本类型
- 标题/菜单：简洁，避免句号；示例：`Looks` → `外观`。
- 对话正文：允许换行；尽量保持英文分句对齐以便对照维护。
- 长文案：将英文完整字符串作为键，置于语言文件；避免在代码中动态拼接。

## 启动与加载
- 语言脚本是异步的，UI 在加载期间可能先显示英文；需要延后早期弹窗到语言设置完成之后。
- 规则：若检测到目标语言为非英文或字典未就绪，则推迟调用弹窗，待 `reflectLanguage()` 完成后再显示。

## 专业术语
- halo → 高亮/光环（根据上下文）
- stage → 舞台
- palette → 调色板
- sprite → 角色
- blocks → 积木

## 审核与发布
- 通过 `audit.js` 生成缺失与占位符差异报告。
- 通过 `extract_apply.js` 生成并应用候选；先在分支预览验证，再提交。
